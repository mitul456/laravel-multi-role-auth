🚀 Installation

Step 1: Install via Composer

composer require mitul456/laravel-multi-role-auth

Step 2: Run Installation Command

php artisan multirole:install

During installation, you'll be prompted to seed default roles. Select yes

Step 3: Add Trait to User Model

Open app/Models/User.php and add:

use Mitul456\LaravelMultiRoleAuth\Traits\HasRoles;

class User extends Authenticatable
{
    use HasRoles;
    // ... rest of your model
}

Step 4: Run Migrations

php artisan migrate

⚙️ Configuration

Publish config file for customization:

php artisan vendor:publish --tag=multirole-config

Edit config/multirole.php:

return [
    'default_role' => 'User',  // Default role for new users
    'role_hierarchy' => [
        'SuperAdmin',
        'Admin', 
        'Moderator',
        'Editor',
        'User',
    ],
    'redirect_paths' => [
        'SuperAdmin' => '/superadmin/dashboard',
        'Admin' => '/admin/dashboard',
        'default' => '/dashboard',
    ],
];

👥 Role Management

Create Role

use Mitul456\LaravelMultiRoleAuth\Models\Role;

$role = Role::create([
    'name' => 'Manager',
    'guard_name' => 'web',
    'description' => 'Manages team',
    'priority' => 50
]);

Assign Role to User

$user->assignRole('Admin');
$user->assignRole(['Editor', 'Moderator']); // Multiple roles
$user->syncRoles(['Editor']); // Replace all roles

Check User Role

if ($user->hasRole('Admin')) {
    // User is admin
}

if ($user->hasRole(['Admin', 'SuperAdmin'])) {
    // User has either role
}

$primaryRole = $user->getPrimaryRole(); // Get highest priority role

🔐 Permission Management

Create Permission

use Mitul456\LaravelMultiRoleAuth\Models\Permission;

$permission = Permission::create([
    'name' => 'edit-articles',
    'guard_name' => 'web',
    'description' => 'Can edit articles'
]);

Assign Permission to Role

$role = Role::where('name', 'Editor')->first();
$role->givePermissionTo('edit-articles');
$role->syncPermissions(['edit-articles', 'delete-articles']);

Check Permission

if ($user->can('edit-articles')) {
    // User can edit articles
}

🛡️ Middleware Protection

// Single role required
Route::get('/admin', function() {
    return view('admin.dashboard');
})->middleware('role:Admin');

// Multiple roles allowed (any)
Route::get('/moderator', function() {
    return view('moderator.area');
})->middleware('role:Moderator|Admin');

// Route group
Route::middleware(['auth', 'role:Admin'])->prefix('admin')->group(function () {
    Route::get('/users', [UserController::class, 'index']);
    Route::get('/roles', [RoleController::class, 'index']);
});

⌨️ Artisan Commands

Install Package

php artisan multirole:install

Create Role

php artisan role:create "Manager" web --description="Manages team"

Assign Role to User

php artisan role:assign 1 "Admin"

Sync User Roles

php artisan role:sync 1 "Editor" "Moderator"

📱 API Middleware

// In routes/api.php
Route::middleware(['auth:api', 'role.api:Admin'])->group(function () {
    Route::get('/admin/stats', [StatsController::class, 'index']);
    Route::post('/admin/users', [UserController::class, 'store']);
});

// Error response for unauthorized
{
    "error": "Forbidden",
    "message": "Role 'Admin' is required"
}

🎯 Dynamic Roles

Create unlimited roles at runtime!

// Create role on the fly
$role = Role::create([
    'name' => 'DepartmentManager',
    'guard_name' => 'web',
    'priority' => 45
]);

// Assign dynamically created role
$user->assignRole('DepartmentManager');

// Check dynamic role
if ($user->hasRole('DepartmentManager')) {
    // Works perfectly!
}

🔧 Troubleshooting

Role methods not found

Add HasRoles trait to User model

use Mitul456\LaravelMultiRoleAuth\Traits\HasRoles;

Middleware not working

Clear route cache

php artisan route:clear

Class not found

Run composer dump-autoload

composer dump-autoload
